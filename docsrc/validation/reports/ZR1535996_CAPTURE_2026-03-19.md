## 4. Practical Conclusion

- Direct normal-device access to `ZR1535996` is operational in this capture.
- The command pair is standard direct word access:
  - read: `0401/0000`
  - write: `1401/0000`
- This proves that the previously validated `ZR163839` boundary is not a protocol-wide or client-wide hard limit.
- It is target-specific behavior from the earlier validated iQ-R environment.

## 5. Repository Impact

- No library code change is required for this capture alone.
- The current client can already encode `ZR1535996`:
  - Q/L format limit: `0xFFFFFF`
  - iQ-R format limit: `0xFFFFFFFF`
- The correct interpretation is:
  - client-side encoding supports large `ZR` values
  - actual acceptance still depends on the target PLC configuration and series-specific environment
